# Sub-agent B Implementation Checklist v0.1

Status: integrated

## Acceptance Criteria Adopted

- `gee-skill` exposes `search-docs`, `plan`, `validate`, `smoke-test`, `run`, and `monitor-exports`.
- Offline commands do not import or initialize Earth Engine.
- Retrieval is deterministic and returns source metadata, score, and excerpt.
- Templates render with strict undefined variables and reject path traversal.
- Static validation parses Python with `ast` and reports `error`, `warning`, and `info`.
- `run` refuses live execution without passing validation and an explicit project.
- `monitor-exports` has timeout and JSON output.
- Unit tests cover retrieval, templates, validation, and CLI offline behavior.

## Risks Addressed

- Earth Engine imports are lazy.
- Offline tests do not require credentials.
- Generated scripts are validated before live execution.
- Local docs index is generated from curated markdown rather than live web access.

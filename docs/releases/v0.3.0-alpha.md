# v0.3.0-alpha Release Notes

Date: 2026-07-01

## Summary

`gee-agent-skill` v0.3.0-alpha frames the repository as a general, agent-native Google Earth Engine harness. Public v0.1/v0.2 workflows are retained as compact golden regression examples; private academic demos are intentionally withheld from public docs, examples, evidence bundles, and packaged resources.

## What Changed

- Added a broader CLI contract for `auth`, `catalog`, `aoi`, `recipe`, `plan`, `render`, `validate`, `preflight`, `run`, `exports`, `trace`, `corpus`, and `eval`.
- Added editable `gee-plan/v0.3` plans and schema validation.
- Added recipe registry and packaged resource fallbacks.
- Added dataset, operator, recipe, failure, and workflow cards for RAG-grounded planning.
- Added static and semantic validation for approved Earth Engine Python templates.
- Added generic v0.3 preflight gates that block placeholder context before export.
- Added benchmark coverage for supported, ambiguous, unsupported, and safety-critical task cases.
- Added documentation for the closed learning loop and tool permission boundaries.

## Benchmark Result

Release checks should run:

```bash
gee-skill eval evals/benchmark_suite.yml --json
```

The benchmark tests harness behavior, not full AutoGEEval-scale Earth Engine code-generation coverage.

## Live-Verified Scope

Live-verified means live preflight passed, one explicit confirmed export was submitted, the task was monitored, and completion evidence was recorded. This scope is limited to public golden examples listed in `docs/capability_matrix.md`.

Non-golden workflows must be described as plan-only, render/validate, mocked preflight, or planned until they receive recipe-specific live preflight adapters and live export evidence.

## Limitations

- The parser is deterministic and pattern-oriented.
- Scientific interpretation requires domain review beyond the harness.
- Users must provide Earth Engine access, Google Cloud Project, local OAuth credentials, quota, and export destinations.
- The repository does not include credentials, OAuth tokens, service account JSON, refresh tokens, private keys, credential paths, private asset ids, unpublished results, or private academic demos.
- The knowledge base is a distilled local corpus and does not replace current official Earth Engine documentation.

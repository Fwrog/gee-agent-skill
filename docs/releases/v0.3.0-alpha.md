# v0.3.0-alpha Release Notes

Date: 2026-06-25

## Summary

`gee-agent-skill` v0.3.0-alpha reframes the repository from a Hong Kong NDVI demo into a general, agent-native Google Earth Engine harness. The Hong Kong workflows are retained as golden examples and regression evidence.

## What Changed

- Added a broader CLI contract for `auth`, `catalog`, `aoi`, `recipe`, `plan`, `render`, `validate`, `preflight`, `run`, `exports`, `trace`, `corpus`, and `eval`.
- Added editable `gee-plan/v0.3` plans and schema validation.
- Added recipe registry and packaged resource fallbacks for wheel installs.
- Added dataset, operator, recipe, failure, and research knowledge cards for RAG-grounded planning.
- Added static and semantic validation for approved Earth Engine Python templates.
- Added generic v0.3 preflight gates that block placeholder context before export.
- Added benchmark coverage for supported, ambiguous, unsupported, and safety-critical task cases.
- Added sanitized golden evidence bundles and public capability status documentation.

## Benchmark Result

Release-prep evidence recorded 22/22 benchmark cases passing with:

```bash
gee-skill eval evals/benchmark_suite.yml --json
```

The benchmark is intentionally compact. It tests harness behavior, not full AutoGEEval-scale Earth Engine code-generation coverage.

## Packaging Result

Release-prep evidence recorded successful source distribution and wheel builds:

```bash
python -m build --sdist --wheel
```

Wheel smoke tests verified package-resource access for `info`, `recipe list`, `catalog evidence`, and `plan from-text`.

## Live-Verified Scope

Live-verified means live preflight passed, one explicit confirmed export was submitted, the task was monitored, and completion evidence was recorded. This scope is limited to the golden examples in `docs/capability_matrix.md`.

Non-golden workflows must be described as plan-only, render/validate, mocked preflight, or planned until they receive recipe-specific live preflight adapters and live export evidence.

## Limitations

- The parser is deterministic and pattern-oriented, not full natural-language understanding.
- Scientific interpretation requires domain review beyond the harness.
- Users must provide Earth Engine access, Google Cloud Project, local OAuth credentials, quota, and export destinations.
- The repository does not include credentials, OAuth tokens, service account JSON, refresh tokens, private keys, or credential paths.
- The knowledge base is a distilled local corpus and does not replace current official Earth Engine documentation.
